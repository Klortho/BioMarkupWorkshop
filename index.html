E-utilities

  • Entrez Programming Utilities
  • NCBI has this many databases (51, at the time I write this)
  • They are all linked to together with the Entrez links database
  • Example: this article → pmc_taxonomy → canis lupus (and 19 others). (Interestingly, not canis lupus familiaris).

E-links

  • Entrez links - list
  • A lot like RDF predicates!
  • But not quite LOD:
    • Identfiers don't have URIs, which means they aren't dereferenceable
    • And can't be referred to by others (in an unambiguous machine readable way)
    • Can only be used to link NCBI databases to NCBI databases
    • Can't declare properties on these (to define, for example, reciprocal links)
    • Can't support SPARQL federated queries

NCBI RDF URI Standards

  • To convert elinks to RDF, this is a necessary first step
  • NCBI RDF URI Standards
  • Uses the new, dedicated http://rdf.ncbi.nlm.nih.gov/ base URI
  • RDF URI Resolver project
  • PubChem RDF just released - using this ID scheme
